## Migration Path from Lovable

If you want to completely leave Lovable:

### Step 1: Export Data
```sql
-- Dump your database
pg_dump $SUPABASE_URL > modelmix_backup.sql
```

### Step 2: Set Up New Infrastructure
```bash
# Option A: Vercel + Supabase Cloud
vercel deploy
supabase link --project-ref your-new-project

# Option B: Self-hosted
docker-compose up -d supabase
npm run build
nginx -c nginx.conf
```

### Step 3: Import Data
```sql
psql $NEW_DATABASE_URL < modelmix_backup.sql
```

### Step 4: Update Environment Variables
```bash
# Update .env
VITE_SUPABASE_URL=https://your-new-project.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-new-key

# Update edge function secrets
supabase secrets set OPENAI_API_KEY=...
```

### Step 5: Deploy
```bash
vercel --prod
# or
docker-compose restart
```

**Estimated time:** 2-4 hours

- [ ] Check AI provider terms (OpenAI, Anthropic, etc.)
- [ ] Set up your own privacy policy
- [ ] Configure GDPR compliance if EU users
- [ ] Set up terms of service
- [ ] Implement cookie consent (already in code)

### Third-Party API Terms to Review:
- [ ] OpenAI: https://openai.com/policies/terms-of-use
- [ ] Anthropic: https://www.anthropic.com/legal/commercial-terms
- [ ] Google: https://ai.google.dev/gemini-api/terms
- [ ] OpenRouter: https://openrouter.ai/terms

**Key restrictions:**
- Most providers prohibit building competing AI APIs
- Must attribute AI providers in your UI (optional but recommended)
- Rate limits apply (check each provider's limits)
